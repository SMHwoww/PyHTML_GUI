# AI对话功能需求规格说明书

## 1. 概述

### 1.1 目标

为 PyHTML 项目添加 AI 对话功能，使用户能够通过自然语言描述需求，AI 自动生成完整的 .pyhtml 格式的网页项目配置，可直接在 PyHTML 中加载使用。

### 1.2 背景

PyHTML 是一个组件化的 HTML 生成器，支持通过拖拽组件来构建网页。通过集成阿里云百炼大模型，可以让用户通过自然语言快速生成网页项目配置，提高开发效率。

## 2. 功能需求

### 2.1 核心功能

- **AI 对话界面**：在 GUI 中添加 AI 对话面板
- **API 密钥配置**：支持配置阿里云百炼 API Key
- **项目生成**：根据用户需求生成完整的 .pyhtml 格式项目
- **项目预览与加载**：生成的项目可以直接预览或加载到编辑器中

### 2.2 详细需求

#### 2.2.1 AI 对话面板

**功能描述：用户通过对话面板与 AI 进行交互

需求：
- 对话历史记录
- 输入框用于描述需求
- 发送按钮
- 加载状态显示
- 错误提示

#### 2.2.2 API 配置面板

**功能描述**：配置阿里云百炼 API 配置

需求：
- API Key 输入框
- 模型选择（默认使用 qwen-max）
- 保存/测试连接功能
- 配置持久化存储

#### 2.2.3 项目生成

**功能描述**：AI 根据用户需求生成 .pyhtml 项目

需求：
- 提示词工程，包含组件列表信息
- 输出格式严格符合 .pyhtml 规范
- 支持验证生成结果
- 生成后自动加载或保存

## 3. 非功能需求

### 3.1 性能需求

- API 请求响应时间 < 30 秒
- 界面响应流畅，无明显卡顿

### 3.2 兼容性

- 错误处理友好，提供明确的错误提示
- 支持离线模式（无 API 时功能禁用但不影响其他功能）

### 3.3 可扩展性

- 模块化设计，便于后续添加其他 AI 提供商
- 提示词可配置

## 4. 系统架构设计

### 4.1 模块划分

| 模块 | 文件路径 | 职责 |
| --- | --- | --- |
| AI 客户端 | src/core/ai_client.py | 封装阿里云百炼 API 调用 |
| 提示词生成器 | src/core/prompt_generator.py | 生成 AI 提示词，包含组件信息和格式要求 |
| GUI 界面 | src/gui/ai_dialog.py | AI 对话面板和配置面板 |
| 集成到主窗口 | src/gui/main_window.py | 集成 AI 功能到主界面 |

### 4.2 数据流程

1. 用户输入需求 → 构建提示词 → 调用阿里云百炼 API → 解析 .pyhtml → 验证格式 → 加载到项目 → 预览

## 5. 验收标准

### 5.1 功能验收

- [ ] 能够成功配置 API Key
- [ ] 能够正常与 AI 对话
- [ ] AI 能根据需求生成合理的项目
- [ ] 生成的 .pyhtml 文件符合规范
- [ ] 生成的项目可以正常加载和预览
- [ ] 错误处理友好

### 5.2 代码质量

- [ ] 代码符合项目现有风格
- [ ] 有必要的错误处理
- [ ] 文档齐全

## 6. 风险与应对

| 风险 | 影响 | 应对策略 |
| --- | --- | --- |
| API 调用失败 | 高 | 提供友好错误提示，支持重试 |
| AI 生成格式错误 | 中 | 添加格式验证和提示修正 |
| API Key 泄露 | 高 | 本地加密存储，不提交到版本控制 |
