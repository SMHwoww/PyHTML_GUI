# 项目文件结构说明

本文档详细说明了PyHTML项目的文件结构，帮助开发者理解项目的组织方式。

## 根目录结构

```
PyHTML_GUI/
├── asset/                # 静态资源文件
├── docs/                 # 项目文档
│   ├── 文件结构说明.md    # 文件结构说明
│   ├── 组件开发规范.md    # 组件开发规范文档
│   ├── pyhtml文件格式规范.md  # PyHTML文件格式规范
│   └── 主题设计规范.md    # 主题设计规范
├── src/                  # 源代码目录
│   ├── components/       # 组件库
│   ├── core/             # 核心功能模块
│   ├── gui/              # 图形界面
│   ├── themes/           # 主题文件
│   ├── utils/            # 工具函数
│   ├── __init__.py       # 包初始化文件
│   └── main.py           # 主入口文件
├── .gitignore            # Git忽略文件
├── LICENSE               # 许可证文件
├── README.md             # 项目说明
└── requirements.txt      # 依赖包列表
```

## 详细说明

### asset/

存放项目的静态资源文件，如图标、图片等。

- `icon.png` - 应用程序图标

### docs/

存放项目文档。

- `文件结构说明.md`- 文件结构说明文档
- `组件开发规范.md` - 组件开发规范文档
- `pyhtml文件格式规范.md` - PyHTML文件格式规范
- `主题设计规范.md` - 主题设计规范

### src/

源代码目录，包含项目的所有代码。

#### components/

组件库目录，存放所有内置组件。每个组件是一个独立的目录，包含配置文件、模板和样式。

- `article/` - 文章组件
- `audio/` - 音频组件
- `button/` - 按钮组件
- `card/` - 卡片组件
- `carousel/` - 轮播图组件
- `counter/` - 计数器组件
- `divider/` - 分割线组件
- `form/` - 表单组件
- `headline/` - 标题组件
- `image/` - 图片组件
- `list/` - 列表组件
- `marquee/` - 走马灯组件
- `navigation/` - 导航栏组件
- `progress/` - 进度条组件
- `quote/` - 引用组件
- `tag/` - 标签组件
- `video/` - 视频组件
- `__init__.py` - 包初始化文件

#### core/

核心功能模块，包含项目的核心逻辑。

- `__init__.py` - 包初始化文件
- `ai_client.py` - AI客户端
- `cloudflare_worker.py` - Cloudflare Worker生成器
- `component.py` - 组件类定义
- `generator.py` - HTML生成器
- `image_manager.py` - 图片管理器
- `preview_server.py` - 预览服务器
- `project.py` - 项目管理
- `prompt_generator.py` - 提示词生成器
- `theme.py` - 主题管理

#### gui/

图形界面模块，包含PyQt5实现的GUI。

- `__init__.py` - 包初始化文件
- `ai_dialog.py` - AI对话框
- `image_manager_gui.py` - 图片管理界面
- `main_window.py` - 主窗口类

#### themes/

主题文件目录，存放各种主题配置。

- `blue.json` - 蓝色主题
- `default.json` - 默认主题
- `high_contrast.json` - 高对比度主题
- `purple.json` - 紫色主题

#### utils/

工具函数目录。

- `__init__.py` - 包初始化文件
- `helpers.py` - 辅助函数

#### __init__.py

包初始化文件，使src目录成为一个Python包。

#### main.py

项目主入口文件，负责启动应用程序。

### 其他文件

- `.gitignore` - Git忽略文件
- `LICENSE` - 项目许可证
- `README.md` - 项目说明文档
- `requirements.txt` - 项目依赖包列表

## 组件目录结构

每个组件目录包含以下文件：

```
component_name/
├── config.json       # 组件配置文件
├── template.html     # HTML模板
├── style.css         # CSS样式文件
└── script.js         # JavaScript文件（可选）
```

## 资源文件管理

- 所有静态资源文件应放在 `asset/` 目录中
- 代码中引用资源文件时，应使用相对路径或绝对路径
- 资源文件的路径应在代码中统一管理，便于维护

## 最佳实践

1. 遵循项目的目录结构，将文件放在正确的位置
2. 保持代码的模块化和可维护性
3. 为新组件创建完整的目录结构和文件
4. 定期更新文档，保持文档与代码的同步
