# 组件开发规范

## 目录结构

每个组件应该有以下结构：

```
your_component/
├── config.json       # 组件配置文件（必需）
├── template.html     # HTML模板（必需）
├── style.css         # 样式文件（可选）
└── script.js         # JavaScript文件（可选）
```

## config.json 配置文件

这是组件的核心配置文件，定义了组件的元数据和可配置字段。

### 示例

```json
{
  "name": "marquee",
  "display_name": "走马灯",
  "description": "一个滚动文字组件",
  "version": "1.0.0",
  "author": "Your Name",
  "fields": [
    {
      "name": "text",
      "type": "string",
      "label": "滚动文本",
      "default": "这是一段滚动文字",
      "placeholder": "请输入滚动文本"
    },
    {
      "name": "speed",
      "type": "number",
      "label": "滚动速度",
      "default": 5,
      "min": 1,
      "max": 20,
      "step": 1
    },
    {
      "name": "direction",
      "type": "select",
      "label": "滚动方向",
      "default": "left",
      "options": [
        {"value": "left", "label": "向左"},
        {"value": "right", "label": "向右"},
        {"value": "up", "label": "向上"},
        {"value": "down", "label": "向下"}
      ]
    },
    {
      "name": "color",
      "type": "color",
      "label": "文字颜色",
      "default": "#000000"
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| name | string | 是 | 组件唯一标识（小写字母和下划线） |
| display_name | string | 是 | 显示名称 |
| description | string | 否 | 组件描述 |
| version | string | 否 | 版本号 |
| author | string | 否 | 作者 |
| fields | array | 是 | 可配置字段数组 |

### 字段类型

#### 1. string（字符串）

```json
{
  "name": "text",
  "type": "string",
  "label": "文本内容",
  "default": "",
  "placeholder": "请输入文本"
}
```

#### 2. number（数字）

```json
{
  "name": "speed",
  "type": "number",
  "label": "速度",
  "default": 5,
  "min": 0,
  "max": 100,
  "step": 1
}
```

#### 3. boolean（布尔值）

```json
{
  "name": "show_border",
  "type": "boolean",
  "label": "显示边框",
  "default": false
}
```

#### 4. select（下拉选择）

```json
{
  "name": "align",
  "type": "select",
  "label": "对齐方式",
  "default": "left",
  "options": [
    {"value": "left", "label": "左对齐"},
    {"value": "center", "label": "居中"},
    {"value": "right", "label": "右对齐"}
  ]
}
```

#### 5. color（颜色选择器）

```json
{
  "name": "bg_color",
  "type": "color",
  "label": "背景颜色",
  "default": "#ffffff"
}
```

#### 6. textarea（多行文本）

```json
{
  "name": "content",
  "type": "textarea",
  "label": "文章内容",
  "default": "",
  "rows": 10
}
```

## template.html 模板文件

使用 Jinja2 模板语法，使用 `{{ field_name }}` 引用配置字段。

### 示例

```html
<div class="marquee-component">
  <marquee 
    scrollamount="{{ speed }}" 
    direction="{{ direction }}"
    style="color: {{ color }};"
  >
    {{ text }}
  </marquee>
</div>
```

### 注意事项

- 始终给组件添加唯一的类名前缀，如 `marquee-component`
- 使用 `{{ field_name }}` 语法引用配置字段
- 可以使用 Jinja2 的条件语句和循环语句

## style.css 样式文件

### 示例

```css
.marquee-component {
  padding: 10px;
  background-color: #f0f0f0;
  border-radius: 4px;
}

.marquee-component marquee {
  font-size: 16px;
  font-weight: bold;
}
```

### 注意事项

- 所有样式选择器都应该以组件类名开头
- 避免使用全局样式污染

## script.js JavaScript文件

### 示例

```javascript
document.addEventListener('DOMContentLoaded', function() {
  const marquee = document.querySelector('.marquee-component marquee');
  if (marquee) {
    // 添加交互逻辑
  }
});
```

## 组件加载

组件可以通过两种方式加载：

1. **内置组件**：放入 `src/components/` 目录
2. **第三方组件**：通过GUI界面加载任意目录下的组件

## 最佳实践

1. 组件名称应该具有描述性
2. 提供合理的默认值
3. 保持组件简单，单一职责
4. 文档化组件的使用方法
5. 测试组件在不同配置下的表现
